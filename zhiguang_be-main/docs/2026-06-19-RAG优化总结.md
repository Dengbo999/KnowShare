# RAG 知识问答优化总结

> 日期：2026-06-19 ~ 2026-06-20

## 功能点

### 1. 递归语义切块（MarkdownChunker）

- 6 级递归切分：标题 → 段落 → 句尾换行 → 句中句尾 → 子句 → 窗口兜底
- Token 自适应估算（中文 ~1.5/token，英文 ~4/token）
- 代码块、表格、列表保护不切分
- 章节标题注入 chunk 元数据
- 碎片向 targetTokens（500）合并，重叠在句子边界
- 文件：`MarkdownChunker.java` + `MarkdownChunkerTest.java`（10 用例）

### 2. 查询改写/扩展（QueryRewriter）

- HyDE 假设文档嵌入 + 关键词提取，单次 DeepSeek 调用
- 向量检索用假设答案，关键词检索用提取词
- 失败静默降级到原始查询
- 文件：`QueryRewriter.java` + `QueryRewriterTest.java`（8 用例）

### 3. Reranker 重排序（RerankService）

- DashScope `gte-rerank-v2` Cross-Encoder 精细打分
- RRF 融合 20-60 候选后插入，重排序再 topK 截断
- 复用现有 DashScope API Key
- 失败静默降级到 RRF 排序
- 配置开关：`rag.rerank.enabled`
- 文件：`RerankService.java` + `RerankServiceTest.java`（10 用例）

### 4. 多轮对话记忆（Redis 会话）

- Redis 持久化对话历史，TTL 7 天
- Key: `rag:session:{postId}:{userId}:{sessionId}` 按文章+用户隔离
- 前端 sessionId 存 localStorage，刷新不丢
- 对话气泡 UI + 新建对话按钮
- 会话管理 API：创建/列表/删除
- 匿名用户退化为无状态模式
- 文件：`ConversationStore.java` + `RedisConversationStore.java` + `SessionInfo.java`

### 5. API 变更

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/{id}/qa/stream` | 流式问答（`sessionId` + `scope`，可选登录） |
| `POST` | `/{id}/qa/sessions` | 创建新会话（需登录） |
| `GET` | `/{id}/qa/sessions` | 列出会话列表（需登录） |
| `DELETE` | `/{id}/qa/sessions/{sid}` | 删除会话（需登录） |

### 6. 向量检索 Filter + 检索范围切换

- 向量检索用 Spring AI `filterExpression()` 在 ES 侧过滤，替代 Java 后过滤
- 关键词检索同步改造，`Filter.Expression` 提取 field/value 转 ES term filter
- 索引 metadata 加 `creatorId`，数据来自 `know_posts.creator_id` → `KnowPostDetailRow.creatorId`
- Scope 模式：`single`（单篇知文）| `user`（用户全部知文）
- 前端下拉切换：`[当前知文]` | `[我的知文]`
- 文件：`RagIndexService.java`、`RagQueryService.java`、`QaStreamRequest.java`、`KnowPostRagController.java`

### 7. 前端改造

- EventSource → fetch + ReadableStream + AbortController
- sessionId 管理 + localStorage 持久化
- 对话气泡 UI（用户/AI 区分）
- 新建对话按钮
- 检索范围下拉切换

## 部署注意

- DashScope Rerank API：需在阿里云百炼控制台开通 `gte-rerank-v2` 模型
- Redis：需确保连接可用，会话 key 自动 7 天过期
- application.yml 已通过 .gitignore 排除，部署时手动配置
- 旧 ES 索引数据没有 `creatorId`，user scope 会漏掉：需调用 `POST /{id}/rag/reindex` 重建
